# 铲糖机器人与上位机通讯协议

## 一、概述

1. **协议类型**: TCP/IP协议
2. **角色**: 上位机作为服务端（端口号9090），机器人作为客户端，机器人主动连接上位机
3. **数据格式**: 报文以字符串形式传输，编码：UTF-8
4. **报文格式**: "{"为报文头，"}"为报文尾，报文中所有的符号均为英文状态下的符号

## 二、报文说明

- 报文中有换行符\n
- 重要的数据交互，若是1s内收不到回复，则需要重发

---

## 三、报文类型

### 1. 状态数据

上位机主动发送报文去查询机器人的状态数据。若连续5-10次收不到小车的反馈报文，可认为小车掉线。

#### 上位机 → 机器人

```
{
MessageType=status
}
```

#### 机器人 → 上位机

```
{
MessageType=status
Mode=xxxx              // 模式：自动任务/远程控制/手柄控制
Status=xxxx            // 状态：空闲/运行中
Charge=xxxx            // 电量：0-100表示电池剩余电量0%-100%，精确到小数点后1位
Speed=xxxx             // 速度：精确到小数点后2位，默认单位为m/s
Fault=xxxx             // 故障：机器人当前的故障信息，无故障时xxxx为空
FaultLevel=xxxx        // 故障等级：预警，一般告警，严重告警
TaskId=xxxx            // 任务号：机器人当前执行的任务号
Station=xxxx           // 机器人当前所在的站点编号
Map=xxxx               // 地图：机器人当前所在的地图名，例如：Map=pathMap
X=xxxx                 // 坐标数据：默认单位为mm，无小数点
Y=xxxx
Z=xxxx
A=xxxx                 // 车体角度：默认单位为°，精确到小数点后2位，范围[-180°~180°]
Boom=xxxx              // 轴位置值，浮点数，默认单位为mm
Bucket=xxxx            // 轴位置值，浮点数，默认单位为°
}
```

---

### 2. 任务下发

机器人上同时只能有一条任务。机器人执行完任务后，会自动删除任务，或者上位机给机器人下发取消任务命令后，可以重新给机器人下发新的任务。

#### 上位机 → 机器人

```
{
MessageType=task
TaskId=20251226165835001     // 任务号：2025-12-26 16:58:35生成的001号任务
PickStationId=123              // 取货站点编号
DropStationId=789              // 放货站点编号
}
```

#### 机器人 → 上位机

```
{
MessageType=task
TaskId=20251226165835001
Result=1                       // 1表示任务正确接收
}
```

---

### 3. 任务完成

#### 机器人 → 上位机

```
{
MessageType=taskFinish
TaskId=20251226165835001
}
```

#### 上位机 → 机器人

```
{
MessageType=taskFinish
TaskId=20251226165835001
}
```

---

### 4. 取消任务

当机器人电池电量过低或者气象站数据判断出天气恶劣，且机器人当前有作业任务，上位机主动发送此报文给机器人，取消当前作业任务，然后给其下发返回充电屋的任务指令。

#### 上位机 → 机器人

```
{
MessageType=cancelTask
}
```

#### 机器人 → 上位机

```
{
MessageType=cancelTask
}
```

---

### 5. 命令

#### 上位机 → 机器人

```
{
MessageType=command
Operate=xxxx           // auto：自动任务模式；remote：远程控制模式；reset：故障复位
}
```

#### 机器人 → 上位机

```
{
MessageType=command
Operate=xxxx
}
```

---

### 6. 远程控制

#### 上位机 → 机器人

```
{
MessageType=remoteControl
Robot:upDown=0;leftRight=0.6
Actuator:boom=-0.56;bucket=0
}
```

---

### 7. 相机检测距离

#### 上位机 → 机器人

```
{
MessageType=cameraCheckDistance
Distance=xxxx              // 相机检测的距离值，单位mm
}
```

**说明**:
- 上位机以 >=200ms 的周期发送数据
- **无反馈报文**

---

**Robot（机器人移动，假设速度栏选择的速度为55时：表示手动最大速度的55%）：**
- 点击向上的按钮，则upDown=0.55；点击向下的按钮，则upDown=-0.55；不点击向上/向下按钮时，upDown=0，此轴停止运行
- 点击向左的按钮，则leftRight=0.55；点击向右的按钮，则leftRight=-0.55；不点击向左/向右按钮时，leftRight=0，此轴停止运行

**Actuator（执行机构控制，假设速度栏选择的速度为55时：表示手动最大速度的55%）：**
- boomUp=0.55表示以55%的速度伸出；boom=-0.55表示以55%的速度缩回；boom=0表示此轴停止运行
- bucket=0.55表示以55%的速度上翻；bucket=-0.55表示以55%的速度下翻；bucket=0表示此轴停止运行

#### 机器人 → 上位机

**无回复报文**

**安全机制**：
若机器人在被上位机远程控制运行时，500ms内收不到上位机发送给机器人的报文，机器人将自动停止运行，以防网络问题导致的机器人不能停止的问题。所以在上位机界面，当运行的按钮被一直按下时，需要连续不断（间隔80-100ms）的给机器人发送报文指令，按钮松开后也应立即发送一条各轴为0的报文指令，机器人收到后会立即停止动作。

---

## 四、状态机转换

```
[空闲] --任务下发--> [运行中] --任务完成--> [空闲]
   |                      |
   |--远程控制模式-------|--紧急停止--> [故障]
   |                      |                |
   |--取消任务-----------|                |--故障复位
   |                      |                |
   +----------------------+----------------+
```

## 五、安全机制

1. **超时保护**：500ms未收到指令，机器人自动停止
2. **心跳检测**：状态查询连续5-10次无响应，判定掉线
3. **重发机制**：重要数据1s内未收到回复，需要重发
4. **紧急停止**：远程控制模式下，按钮松开后立即发送停止指令
